    VuePress + Github Pages 搭建博客

    Neko 2023/9/8 VuePress

    本文主要介绍如何使用 VuePress 搭建个人博客并部署到 Github Pages，以及 vuepress-theme-reco 主题美化和插件等扩展功能的使用。

    VuePress 官方文档 (opens new window) 、vuepress-theme-reco 官方文档 (opens new window)

    # 前言

    VuePress 是一个以 Markdown 为中心的静态网站生成器。
    如果想快速搭建一个静态的博客网站来简单记录记录笔记或者文章，可以使用 Markdown 来书写内容（如文档、博客等），然后 VuePress 会帮助我们生成一个静态网站来展示它们。 而且上手非常简单，新手小白也可以分分钟搭建出属于自己的博客。

    # 目录结构

    —— blog
  |—— docs
      |—— .vuepress     // 配置目录
          |—— public    // 静态资源
          |—— styles    // 样式
          |—— config.js     // 配置文件
          |—— enhanceApp.js // 客户端应用增强文件
      |—— notes     // markdown笔记文档
      |—— README.md     // 博客首页

    # 1.本地搭建

    1.创建并进入一个新目录

    mkdir vuepress-starter
cd vuepress-starter
    1
    2

    2.使用 npm 或者 yarn 进行初始化，我这里使用的是 npm 包管理器。

    npm init -y
    1

    3.安装VuePress为本地依赖

    npm install -D vuepress
    1

    4.创建第一篇文档
    执行以下命令会先创建 docs 文件夹，然后在文件夹下创建 README.md 文件，写入的内容为 Hello Vuepress。

    mkdir docs
echo '# Hello VuePress' > docs/README.md
    1
    2

    注意：使用命令创建的话，可能会出现文字乱码情况，把 README.md 的编码格式改为UTF-8即可

    5.在 package.json 中的 scripts 里添加如下代码。

    "scripts": {
  "docs:dev": "vuepress dev docs",
  "docs:build": "vuepress build docs"
}
    1
    2
    3
    4

    6.在终端执行以下命令，启动服务器，端口为 8080 http://localhost:8080

    npm run docs:dev
    1

    # 2.基础配置

    # 2.1. 添加标题和描述

    1.在 docs 文件夹下添加 .vuepress 目录，所有 VuePress 相关的文件都会被放在这里。
    在.vuepress 文件夹下添加 config.js，配置网站的标题和描述，此时目录结构可能是这样：

      |—— docs
      |—— README.md
      |—— .vuepress
          |—— config.js

    # 3.更换主题

    目前为止博客的基本功能已经实现。
    接下来使用主题对界面美化并添加一些动画效果和便捷功能，我使用的主题是 vuepress-theme-reco

    # 3.1.安装主题

    npm install vuepress-theme-reco --save-dev
    1

    然后在 config.js 中引用该主题：

    module.exports = {
  title: 'XXX的博客',
  description: 'XXXXXX'
  theme: 'reco',
  themeConfig: {...}
}

    # 3.2.修改主题颜色

    VuePress 默认主题颜色是绿色，我们可以将主题颜色修改为自己喜欢的颜色。
    在 .vuepress 目录下创建 styles 文件夹，该文件夹里创建一个 palette.styl 文件，修改主题颜色为蓝紫色，文件代码如下：

    $accentColor = #5d67e8

    # 3.3.其他样式修改

    VuePress 提供了一种添加额外样式的简便方法。可以在之前创建好的 styles 文件夹下再新建一个 index.styl 文件，可以使用正常的 CSS 语法修改样式，例如修改 md 文档里 code 部分内容的文字颜色或背景色，文件代码如下：

    // 代码内容样式
.light .content__default code {
  color: #4a59d7;
}
.dark .content__default code {
  background-color: rgba(58, 58, 92, 0.7);
  color: #fff;
}

    # 3.4.进一步美化

    接下来是对整体的进一步美化，包括导航栏配置，主页头像设置，文章信息等的添加。
    config.js 是项目的全局配置文件，可以配置网站的基本信息、插件、主题、导航栏和侧边栏等内容。头像等图片资源放在了public文件夹中。

    // 完整的 config.js 文件内容，可根据需要自行删减
module.exports = {
  base: "/test/",
  title: "XXX的博客",
  description: "XXXXXX",
  theme: "reco",
  // 移动端优化
  head: [
    ["link", { rel: "icon", href: "/avator.ico" }],
    [
      "meta",
      {
        name: "viewport",
        content: "width=device-width,initial-scale=1,user-scalable=no",
      },
    ],
  ],
  // 设置语言
  locales: {
    "/": {
      lang: "zh-CN",
    },
  },
  markdown: {
    lineNumbers: true, // 代码块显示行号
  },
  themeConfig: {
    // 主题模式 博客模式
    type: "blog",
    autohor: "XXX",
    logo: "/avator.jpg", // 导航栏左侧logo
    authorAvatar: "/avator.jpg", // 个人信息的头像
    // 开启目录结构 子侧边栏
    subSidebar: "auto",
    // 博客配置
    blogConfig: {
      socialLinks: [{ icon: "reco-github", link: "https://github.com/" }],
    },
    nav: [
      { text: "首页", link: "/" },
      {
        text: "XXX的博客",
        items: [
          { text: "知乎", link: "https://www.zhihu.com" },
          { text: "Github", link: "https://github.com" },
        ],
      },
      { text: "TimeLine", link: "/timeline/", icon: "reco-date" },
    ],
    sidebar: {
      "/notes/": [
        {
          title: "文档列表",
          path: "/notes/1",
          collapsable: true, // 是否折叠
          children: [
            { title: "第一篇", path: "/notes/1" },
            { title: "第二篇", path: "/notes/2" },
          ],
        },
      ],
    },
  },
};

    # 5.部署 Github Pages

    博客网站基本搭建完成以后，没有服务器可以将静态博客部署到 Github Pages 上。
    首先在 Github 上新建一个仓库，仓库名自定义，可以取名为blog。相应地，我们需要在 config.js 添加一个 base 路径配置：

    module.exports = {
  ...
  // 和仓库名相同
  base: '/blog/',
  ...
}
    1

    临时效果预览 (无 git 上传 dist 文件)

    如果暂时没有安装 git，可以在vuepress-starter文件夹下，执行npm run docs:build命令，生成docs/.vuepress/dist文件夹，生成好了之后将 dist 文件夹的全部内容，通过 github 上的Add file ->upload files上传到远程仓库上。
    （注：文件过大可能无法上传，或者部分文件名无法识别）

    在项目 docs 目录下创建一个脚本文件 deploy.sh，注意修改一下对应的用户名和仓库名，相关代码如下：

    #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
git push -f git@github.com:XXX/XXX.git master:gh-pages

cd -
